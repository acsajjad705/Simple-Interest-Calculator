# Contributing

**All contributions, bug reports, bug fixes, documentation improvements, enhancements, and ideas are welcome.**

Thank you for your interest in improving this project. This guide explains how to get started, report issues, propose changes, and what we expect from contributors.

---

## How to Contribute

- **Fork** the repository to your GitHub account.
- **Create a branch** for your change (e.g., `fix/readme-typo` or `feature/add-calculator-ui`).
- **Make your changes** and commit with clear messages.
- **Push** your branch and open a **pull request** against the `main` branch.

---

## Reporting Bugs and Suggesting Enhancements

When opening an issue, please include:
- A **clear title** and description.
- Steps to reproduce (for bugs).
- Expected vs actual behavior.
- Environment details (OS, browser, language version).
- Screenshots or logs if helpful.

Use labels like `bug`, `enhancement`, or `documentation` when creating issues.

---

## Pull Request Checklist

Before submitting a pull request:
- [ ] Branch is up to date with `main`.
- [ ] Code follows project style guidelines.
- [ ] Tests pass locally and in CI (if applicable).
- [ ] Documentation is updated if behavior changed.
- [ ] PR description explains the motivation and changes.

---
